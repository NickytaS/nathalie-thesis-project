Help Center

Find answers to frequently asked questions about the Migration Tool Evaluator

What is the Migration Tool Evaluator? ▼
The Migration Tool Evaluator is a decision-support prototype developed as part of a thesis on heterogeneous database migration. It helps database administrators and developers choose the optimal migration tool for their use case. The evaluation is based on real experimental migrations of WordPress, WooCommerce, and ERPNext databases using pgLoader, MongoDB Relational Migrator (MRM), and Mongify — scored across a 35-criteria weighted framework covering schema fidelity, data accuracy, performance, and operational characteristics.
What tools are compared? ▼
Three tools were compared in this thesis:

pgLoader — MySQL → PostgreSQL. Highest overall score (4.83/5.0). Sub-second migration, 100% accuracy on all 3 databases. Note: lowercases all table names in the PostgreSQL target.

MongoDB Relational Migrator (MRM) — MySQL → MongoDB. Score 4.37/5.0. GUI-based, auditable mapping, 100% accuracy, 1–6 seconds. Renames collections to camelCase (wp_posts → wpPosts).

Mongify — MySQL → MongoDB. Score 3.14/5.0. Lightweight CLI tool (Ruby-based). 100% accuracy on 2/3 databases; failed on WordPress due to idempotency bug (appends duplicate documents on re-run). Slowest tool at 15–30 seconds.
How is the recommendation generated? ▼
Recommendations are generated using a weighted compatibility model based on your answers to 6 questions about your migration scenario. The system scores each tool against your stated needs — target database type, schema complexity, importance of relational integrity, speed requirements, interface preference, and document structure priority. The tool with the highest compatibility score is displayed as the recommendation, along with a detailed justification derived from the experimental results.
Can I compare tools manually? ▼
Yes! Click Compare in the navigation to see a side-by-side table of all three tools scored across the 5 evaluation categories with their respective weights: Schema Fidelity (30%), Data Fidelity (30%), MongoDB Transformation (10%), Performance (20%), and Operational (10%). You can also click any tool card on the Home page to see that tool's full score breakdown and experimental justification.
What databases were used for testing? ▼
Three real-world databases of varying complexity were used:

WordPress Blog (blog_db) - 16 tables, 1,448 rows (posts, users, comments, terms, metadata, action scheduler)
WooCommerce E-commerce (ecommerce_db) - 34 tables, 150 rows (orders, products, customers, shipping, payments, analytics)
ERPNext ERP System (erp_db) - 18 tables, 510 rows (users, roles, countries, currencies, languages, contacts)

pgLoader and MRM achieved 100% row count accuracy across all databases. Mongify achieved 100% on ecommerce_db and erp_db, but failed on blog_db due to an idempotency issue (duplicate data from re-running without dropping collections).
