Database Source Files Documentation

Thesis Project — Database Migration Tool Evaluation

Author: Nathalie  |  Date: April 2026

1. Overview

This folder (docker/mysql/) contains the real-world MySQL database files used as source databases for the thesis project. These files were exported directly from a live MySQL 8.0 Docker container after setting up and populating three real-world applications: WordPress, WooCommerce, and ERPNext.

The data was then migrated using three different database migration tools:

  • pgLoader — MySQL to PostgreSQL
  • Mongify — MySQL to MongoDB
  • MRM (MongoDB Relational Migrator) — MySQL to MongoDB

3. How These Files Were Created

The following steps were followed to generate the real-world SQL dump files:

  1. Docker containers were started using docker-compose up -d
  2. WordPress was installed and configured at http://localhost:8080
  3. The WooCommerce plugin was installed; products, customers, and orders were created
  4. ERPNext v14 was set up separately using the frappe_docker environment
  5. All three databases were exported using mysqldump from inside the running container:
docker exec mysql_source mysqldump -u root -prootpassword blog_db       > docker/mysql/blog_db.sql
docker exec mysql_source mysqldump -u root -prootpassword ecommerce_db  > docker/mysql/ecommerce_db.sql
docker exec mysql_source mysqldump -u root -prootpassword erp_db        > docker/mysql/erp_db.sql

These files are mounted into the MySQL Docker container at startup via docker-compose.yml, so the databases are automatically restored whenever the container is re-created.

4. Why Real-World Data Was Used

Important thesis justification: Using real application data instead of synthetic or randomly generated data ensures that the migration test results accurately reflect how these tools perform in production-like scenarios.

Key benefits of using real data:

  • Authentic table structures with real column types, constraints, and relationships
  • Realistic data volumes and content (no artificially simplified schemas)
  • Valid testing of tool behaviour on complex, real-world schemas
  • Credible and reproducible thesis results
  • Ability to detect tool-specific failure modes (e.g., Mongify's idempotency issue)
